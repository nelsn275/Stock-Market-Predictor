PredictAStock — Quick Guide

A simple, static-first UI showing stock price data and model predictions.

What this site shows

The page displays a watchlist of stocks with their latest Open and Close prices, a computed Change, and a model prediction for a selected time horizon (1 day, 1 month, 1 year, 5 years). The prediction for each stock is one of: BUY, SELL, or HOLD.

How to use it (for anyone)

  1. Open index.html in your browser. For best results, serve the folder with a simple static server (instructions below).
  2. The page looks for a file in the json/ folder called latest_with_preds.json or stocks.json. If found, it will load the data automatically.
  3. Use the Timeframe buttons above the table to switch which prediction you see (1d/1m/1y/5y).
  4. Click the small reload icon to re-read the JSON files if you updated them on disk.
  5. Use the Night switch in the top-right to toggle dark mode.

Color legend

BUY (green)
SELL (red)
HOLD (amber)

The Change column is green when the Close is higher than the Open, red when lower, and gray when unchanged or not available.

How this works (simple explanation)

A model was trained on historical market and economic data to produce predictions for many stocks. The predictions are saved to a JSON file (called results.json), and the latest prices are fetched and merged with the predictions into files inside the json/ folder. The website simply reads those JSON files and shows the results — no server required.

For maintainers / curious users

If you want to regenerate the JSON used by the site: 

python stocks.py
# or (if you need to retrain models) python decision_tree.py

If you don't want to run Python locally, you can simply drop a file named latest_with_preds.json inside the json/ folder that follows the same format (an array of objects with keys like Ticker, Open, Close, Pred_1d, etc.).

Troubleshooting

  • If the page shows "Loading stock data…" but never loads, make sure you are serving the files over http:// (browsers often block local file fetches). Run: python -m http.server 8000 and open http://localhost:8000/index.html.
  • If numbers look wrong, confirm the JSON files in the json/ folder contain the expected fields (`Open`, `Close`, `Pred_1d` / `Action_1d`, etc.).

Want to help improve this?

Suggestions: add a schedule to regenerate the JSON automatically, host the UI as a static site (Netlify/GitHub Pages), or improve the prediction confidence display (show probability or model score next to BUY/SELL).
